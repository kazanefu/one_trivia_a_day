非同期処理について①

長くなりすぎるのもよくないと思ったので, 今日は非同期処理とはどういうものかだけ説明して, 具体的にどのようなときに使うのか, どうやって使うのかはいずれ次回説明します.

**並行処理と並列処理の違い**

並行処理はスレッドを占有せずに処理をしていないときは他のタスクにスレッドを譲る

並列処理は複数のスレッドで同時に処理を行う

task1 = 1, task2 = 2を同時にしたいとき
並行処理:
```
|1|2|1|1|2|2|
```
並列処理
```
|1| |1|1| |
|2| | |2|2|
```
こんな感じで, task1が止まっている間にそのスレッドでtask2を処理できるのが並行処理, task1とtask2を別スレッドで同時に処理するのが並列処理です.

非同期処理はうまく使えば実行速度を大幅に向上させることができることもありますが, 無暗に意味のないところで使っても恩恵はありませんしむしろオーバーヘッドが発生するので, どのようなときに非同期処理を使う必要があるのかをよく考えて使いましょう.

ちなみに, AtCoderなどでは並列処理をしたとしてもそれぞれのスレッドでの時間の合計が時間として計算されるため, 並列処理をしても意味のないです。またタスクが止まることがあるような処理もないので, 並行処理も意味がないです.
