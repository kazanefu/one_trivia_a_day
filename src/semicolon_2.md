# 今日の豆知識.2
セミコロン`;`の意味

言語によるセミコロンの有無は文法の違いというより意味論の違いがあります

セミコロンがある言語の代表例
- C/C++
- C#
- Java
- JavaScript
- Go（※自動挿入ありだが意味はある）
- Rust <- セミコロンの意味が挙動にそのまま出るのでわかりやすい

セミコロンがあまり意味を持たないorそもそもない言語の代表例
- Python
- Ruby
- Haskell
- Lua

セミコロンが意味を持つ言語では
```rust
let x = 10;
println!("{}",x);
```
や
```c
int x = 10;
printf("%d\n",x);
```
のように`;`は**ここで1つの文（statement）が終わる** という意味的な区切りであり,「文」という概念が言語の中心にある
また,Rustでは`;`の意味が特に分かりやすく
```rust
let x = {
    1
};
```
と
```rust
let x = {
    1;
};
```
では意味が異なり,前者は右辺の値は`1`であるが,後者では`()`つまり他の言語での`void`に相当する
これは`;`の有無によって**式**であるか**文**であるかの意味付けをしているからであり,前者は`1`という**式**であり値を持つのに対して後者は**1;**という文であるため値は持たない

`;`があまり意味を持たない言語も二つに分けられる
例えばPythonでは文の区切りは**改行**によって行われていてセミコロンは同一行上に複数の文を書くための構文上の糖衣になっている
```python
x = 1; y = 2 # 推奨はされていない
```
Pythonなどの言語では`;`はなくとも同じ働きをするもの(改行など)が存在する. ただし
```python
transform = transforms.Compose([
    transforms.Grayscale(num_output_channels=1),
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
])
```
のように複数行にわたって一つの文を書くこともあるため改行が文の区切りとしての意味を持たないときもある.

一方,Haskellのような**式(expression)**中心の言語ではそもそも分の区切りが不要だったりします(僕がHaskell,圏論ともにエアプなので詳しい説明はしないでおきます)
簡単に言うと
```haskell
putStrLn "A" >> putStrLn "B"
```
はpythonでの
```python
print('A')
print('B')
```
と同じ動作をしますが
`putStrLn "A"`も`putStrLn "B"`も文ではなく式です(詳しくはちゃんと圏論を理解した後にいずれ説明します)
